# 北冥服务器官网需求文档 v5.0

## 1. 文档信息

- 项目名称：北冥服务器官网
- 文档版本：v5.0
- 编写日期：2026-04-29
- 项目定位：面向 Minecraft 服务器的官网、入服审核、社区入口、内容展示、贡献考勤与运营管理平台
- 技术约束：后端使用 JDK 8 + Spring Boot，前端使用 React，数据库使用 MySQL

## 2. 项目背景

北冥服务器需要建设一个官方站点，用于承载服务器宣传、玩家入服、指南浏览、资源分发、服务器状态查询、成员展示、素材沉淀、贡献考勤和后台运营管理。

服务器日常交流主要发生在 Oopz 软件、QQ群聊天和游戏内聊天。官网不替代这些即时交流渠道，而是作为统一入口、规则说明、正式流程、内容归档和管理后台。玩家需要通过官网了解服务器、完成入服审核、查看指南和资源；管理员需要通过官网管理账号、白名单、素材审核、考勤积分、工单举报和运营数据。

## 3. 产品目标

## 3.1 核心目标

- 建立统一官网入口，展示服务器定位、状态、规则、指南、资源、成员和入服流程。
- 建立账号体系，支持邀请码注册、登录、Minecraft ID 绑定和四级权限管理。
- 建立入服审核体系，支持红石、后期、建筑、其他方向的入服考试和白名单审核。
- 建立外部交流入口，清晰引导玩家前往 Oopz、QQ群和游戏内聊天参与即时交流。
- 建立正式处理渠道，支持论坛沉淀、工单申诉、玩家举报、站内通知和消息中心。
- 建立内容平台，展示摄影素材、成员作品、精彩活动、美好瞬间、服务器进度、成就和专题合集。
- 建立贡献与考勤体系，记录玩家活跃度、工程贡献、活动参与、考勤积分和榜单。
- 建立管理后台，集中管理用户、邀请码、指南、审核、素材、资源、工单、举报、榜单、考勤和系统配置。

## 3.2 第一阶段目标

- 游客可以浏览首页、服务器状态、服规、进服指南、游玩指南、外部交流入口、成员列表、精选内容和精彩瞬间。
- 玩家可以通过邀请码注册登录、绑定 Minecraft ID、选择审核方向、完成入服考试、提交白名单申请。
- 玩家可以上传摄影、作品、精彩瞬间等素材，审核通过后公开展示。
- 管理员可以维护公告、指南、邀请码、题库、白名单审核、素材审核、考勤积分和基础配置。
- 官网仅展示 Oopz、QQ群、游戏内聊天入口和用途说明，不做自动消息同步、聊天记录保存或跨平台 SSO。

## 4. 用户角色

## 4.1 游客

未登录用户。可以浏览公开内容，包括首页、服务器状态、服规、指南、公告、外部交流入口、成员列表、精选作品、精彩瞬间、公开榜单和公开资源。

## 4.2 注册玩家

已通过邀请码注册的用户。可以绑定 Minecraft ID、参加入服审核、提交白名单申请、参与论坛、投票、上传素材、提交作品、提交工单和举报、查看个人中心和消息中心。

## 4.3 见习成员

通过“其他/普通/见习”方向入服或处于观察期的成员。可以进入服务器，但部分权限、工程参与资格或资源访问范围可低于正式成员。

## 4.4 正式成员

已通过审核并加入服务器的玩家。可以访问成员资源、参与成员投票、展示成员名片、成员事迹、作品、活动记录、工程贡献、活跃度和考勤积分。

## 4.5 管理员与协管

负责日常运营。按 `owner`、`admin`、`helper`、`user` 四级权限执行不同管理能力。

## 5. 功能模块

## 5.1 官网展示与品牌模块

### 功能说明

官网首页是服务器对外入口，用于展示服务器定位、当前状态、入服路径、精选内容和常用入口。

### 功能点

- 展示服务器名称、简介、玩法类型、版本、进服入口和服务器定位。
- 展示服务器实时状态，包括在线状态、在线人数、最大人数、延迟、MOTD 和版本。
- 展示开服至今累计运行时间。
- 展示最新公告、开放活动、现役工程、版本更新和维护通知。
- 展示精选摄影、成员作品、精彩瞬间、成员事迹、服务器进度和专题合集。
- 展示玩家活跃度榜单、业绩考勤榜单的简版入口。
- 展示 Oopz、QQ群、游戏内聊天等外部交流入口。
- 提供注册登录、进服指南、入服考核、白名单申请、资源下载、论坛、工单入口。
- 支持 SEO 标题、描述、关键词、分享图、站点图标和品牌素材配置。

### 验收标准

- 用户进入首页后，可以快速理解服务器是什么、是否在线、如何加入、去哪里交流。
- 首页核心数据异常时，需要显示友好降级提示，不能整页空白。
- 首页展示内容应由后台配置，不能写死在前端代码中。

## 5.2 用户、权限与邀请码模块

### 功能说明

负责官网账号、登录状态、邀请码、角色权限和 Minecraft 身份绑定。用 Java 后端语境讲，这个模块就是“用户服务”，控制谁能登录、谁有什么权限、谁绑定了哪个游戏 ID。

### 功能点

- 用户注册、登录、退出登录。
- 邀请码注册，区分玩家邀请码和管理员邀请码。
- 邀请码支持有效期、使用次数、绑定角色、禁用状态和使用记录。
- 管理员邀请码只能由 owner 创建和分发。
- 找回密码、重置密码、邮箱验证或验证码。
- Minecraft ID / UUID 绑定。
- 用户资料维护，包括头像、昵称、简介、联系方式、语言偏好、主题偏好。
- 管理系统权限固定分为四级：`owner`、`admin`、`helper`、`user`。

### 权限等级

| 等级 | 定位 | 权限范围 |
| --- | --- | --- |
| owner | 服主 / 超级管理员 | 拥有全部权限，可管理系统配置、管理员邀请码、角色分配、考勤规则、白名单移除、数据备份、外部服务配置和所有高风险操作。 |
| admin | 管理员 | 负责日常运营，可管理用户、公告、指南、活动、题库、白名单审核、素材审核、论坛、举报、工单、资源、榜单和数据看板，但不能修改 owner 权限、不能创建 owner、不能绕过审计日志。 |
| helper | 协管 / 审核员 | 负责辅助审核和社区处理，可处理普通工单、初审举报、初审素材、维护部分内容、回复玩家问题，但不能移除白名单、不能手动大额扣分、不能生成管理员邀请码、不能修改系统配置。 |
| user | 普通用户 / 玩家 | 可浏览公开内容、注册登录、绑定 Minecraft ID、参加入服考核、提交白名单、上传素材、发帖回复、投票、提交工单和举报、查看个人积分与消息。 |

### 验收标准

- 未填写有效邀请码的用户不能完成注册。
- 玩家邀请码不能注册管理员账号。
- 邀请码过期、禁用或超过使用次数时，注册必须失败。
- 同一个 Minecraft ID 原则上只能绑定一个官网账号。
- user 不能访问后台管理接口。
- helper 不能访问 owner/admin 专属接口。
- 高风险操作必须记录操作者、时间、对象、原因和操作前后状态。

## 5.3 入服审核与白名单模块

### 功能说明

考试仅用于入服审核，以及玩家分数不合格、白名单被移除后需要重新入服时的二次考核。考试不作为日常活动、娱乐答题或其他社区功能使用。

### 考核方向

- 红石审核：面向红石、生电、机械、技术类玩家。
- 后期审核：面向具备较强生存后期、资源组织、工程推进能力的玩家。
- 建筑审核：面向建筑、景观、规划、美术表现类玩家。
- 其他审核：面向普通玩家和见习成员。

### 功能点

- 新玩家入服引导：邀请码注册、绑定 Minecraft ID、阅读服规、选择审核方向、完成考试、提交白名单、等待审核、审核通过。
- 在线考试支持单选、多选、判断、简答题。
- 题库按审核方向维护，不同方向使用不同题库和评分规则。
- 客观题自动判分，简答题由管理员人工审核。
- 白名单申请提交 Minecraft ID、审核方向、申请理由、考试结果等信息。
- 白名单审核支持通过、拒绝、要求补充、备注。
- 审核通过后创建成员档案，并将考勤积分初始化为 60 分。
- 分数不合格时允许二次考试入服，二次考试可使用更高难度题库或更高通过分。
- 因考勤积分小于等于 0 被移除白名单的玩家，重新入服必须参加更高难度的二次考核。
- 审核结果通过站内消息通知玩家。

### 验收标准

- 未登录用户不能参加考试。
- 未绑定 Minecraft ID 的用户不能提交白名单申请。
- 考试不能被其他模块当作普通答题功能使用。
- 不同审核方向必须能配置不同题库、通过分和审核规则。
- 二次考核难度必须高于该玩家上次入服考核。
- 管理员可以查看每个玩家的考试历史和白名单审核记录。

## 5.4 指南与知识库模块

### 功能说明

用于集中展示长期有效的官方内容，包括服规、入服流程、游玩说明、成员规范、工程项目和开放活动说明。

### 功能点

- 服规中心：展示游戏规则、社区规则、处罚标准、申诉方式和规则版本。
- 进服指南：说明邀请码注册、绑定 Minecraft ID、考试、白名单申请、选择线路、下载客户端等流程。
- 游玩指南：说明玩法、基础指令、常用插件、领地、传送、经济、资源世界、活动世界等内容。
- 玩家指南：说明成员规范、协作方式、建筑规范、工程参与方式、作品提交规则、贡献加分规则和考勤规则。
- 现役工程项目：展示项目名称、负责人、状态、进度、参与要求、坐标或传送方式。
- 当前开放活动：展示活动时间、报名方式、规则、奖励、参与条件和状态。
- 指南分类、标签、搜索、目录导航和置顶推荐。
- 文档版本记录，重要规则变更需要记录更新时间和修改说明。

### 验收标准

- 游客可以浏览公开指南。
- 成员专属指南只能登录后查看。
- 规则类文档必须展示版本号和最近更新时间。
- 现役工程和开放活动必须有状态标记。
- 管理员可以在后台新增、编辑、发布、下架指南内容。

## 5.5 服务器状态与线路模块

### 功能说明

用于展示服务器是否可用、玩家应该选择哪条线路、每条线路当前连接情况如何。

### 功能点

- 查询主服务器是否在线。
- 查询服务器版本、MOTD、在线人数、最大人数和延迟。
- 配置多条线路，例如主线路、备用线路、加速线路、国际线路。
- 检测每条线路是否可连接。
- 展示每条线路在线人数、延迟和负载情况。
- 记录历史在线人数、峰值人数、宕机记录和线路稳定性。
- 记录开服时间，并计算开服至今的天数、小时数。
- 在线地图暂不实现，只做在线地图接口预留，后续可对接 BlueMap / Dynmap。

### 验收标准

- 状态查询失败时，前端应显示“状态暂不可用”。
- 多线路数据应独立展示。
- 开服时间由后端统一计算。
- 状态查询应有缓存，避免高频探测服务器。
- 在线地图接口预留不在第一阶段展示为正式功能。

## 5.6 外部交流入口模块

### 功能说明

服务器即时交流主要发生在 Oopz 软件、QQ群聊天和游戏内聊天。官网只负责展示入口、用途说明和加入规则，不做聊天同步、不保存聊天记录、不自动转发第三方消息、不做跨平台 SSO。

### 功能点

- Oopz 入口：展示服务器 Oopz 社区或频道链接、用途说明、加入条件、使用规则。
- QQ群入口：展示群号、加群方式、群用途、入群规则、审核要求。
- 游戏内聊天说明：说明游戏内聊天用于即时交流、活动沟通、工程协作、管理提醒。
- 外部交流规则：说明哪些内容适合 Oopz、哪些适合 QQ 群、哪些适合游戏内聊天。
- 官网正式渠道提示：规则、申请、申诉、举报、工单、素材审核和白名单流程以官网记录为准。
- 后台可维护 Oopz 链接、QQ群号、群公告、入群说明和游戏内聊天规则。

### 渠道定位

- Oopz：服务器主要语音、频道、活动组织和成员协作入口。
- QQ群：玩家日常聊天、通知接收、问题咨询和轻量交流入口。
- 游戏内聊天：在线玩家即时沟通、活动指挥、工程协作和管理提醒入口。
- 官网论坛：沉淀教程、建议反馈、公告讨论、作品展示等长期内容。
- 工单与举报：处理正式问题、私密问题、申诉和违规反馈。
- 消息中心：承载官网内部审核、积分、白名单、素材等系统通知。

### 验收标准

- 外部交流入口必须能在首页、帮助中心和加入我们页面找到。
- 后台可以修改 Oopz、QQ群和游戏内聊天说明。
- 文档必须明确官网不做自动聊天同步、聊天记录保存或跨平台 SSO。
- 外部平台链接不可用时，页面需要显示可理解的提示。

## 5.7 社区论坛、工单、举报与消息模块

### 功能说明

该模块负责玩家在官网内的社区沉淀和正式问题处理。论坛用于公开沉淀内容，工单和举报用于正式处理问题，消息中心用于系统通知。

### 功能点

- 论坛分区：公告、玩家交流、建议反馈、BUG 反馈、建筑展示、交易市场、攻略教程。
- 论坛功能：发帖、编辑、删除、回复、点赞、收藏、浏览量统计、置顶、加精、锁定。
- 工单与申诉：处理封禁申诉、白名单问题、账号问题、资源问题、BUG 反馈、素材争议、其他问题。
- 玩家举报：处理帖子、评论、游戏内违规、素材侵权、冒名顶替等问题。
- 违规处罚：记录警告、禁言、封禁、移除白名单、限制投稿、限制发帖。
- 消息中心：通知考试结果、白名单审核、素材审核、活动报名、工单回复、积分变动、处罚通知。
- 支持按用户、角色、全站发送通知。

### 验收标准

- 游客可以浏览公开帖子，但不能发帖和回复。
- 工单内容默认不公开。
- 举报处理必须记录处理人、处理时间、处理结果和说明。
- 关键审核结果必须发送站内消息。
- 用户可以标记消息已读。

## 5.8 内容展示、素材投稿、精彩瞬间与专题模块

### 功能说明

用于展示服务器长期建设成果，沉淀摄影素材、成员作品、精彩活动、美好瞬间、专题合集、服务器进度、成就和里程碑。

### 功能点

- 摄影作品展示，支持图片上传、标题、描述、标签、作者。
- 成员作品展示，支持建筑、红石、地图画、皮肤、视频、剧情作品等类型。
- 精彩活动记录，展示活动名称、时间、地点、参与成员、活动简介、图集和视频。
- 美好瞬间展示，记录有趣瞬间、纪念截图、合影、建筑完成时刻、活动高光时刻。
- 玩家自由投稿，支持上传图片、视频、压缩包、文档、地图素材、宣传素材等文件。
- 素材审核，投稿后进入待审核状态，管理员可通过、拒绝、要求修改、设为精选。
- 素材版权与授权，上传时选择允许官网展示、允许宣传使用、仅成员内部可见等授权方式。
- 内容专题页：将活动、工程、作品、成员、素材和精彩瞬间整理为合集。
- 服务器进度、成就与里程碑展示。

### 验收标准

- 未登录用户不能上传素材。
- 上传素材必须进入审核流程，审核通过前不能公开展示。
- 被拒绝的素材需要向投稿人展示拒绝原因。
- 精选素材可以在首页、精彩瞬间页、专题页和活动详情页展示。
- 上传素材必须记录授权方式。

## 5.9 成员、活跃度、贡献考勤与榜单模块

### 功能说明

用于展示成员信息、成员事迹、玩家活跃情况、工程贡献、活动参与和业绩考勤积分。

### 成员功能

- 成员列表展示，显示成员皮肤、Minecraft ID、昵称、身份组、加入时间、状态。
- 成员详情页，展示个人简介、成员事迹、参与工程、代表作品、活动记录、获得成就、社区贡献、活跃度和考勤积分。
- 成员事迹管理，管理员可维护，玩家本人可提交补充内容。
- 成员皮肤优先通过 Minecraft ID / UUID 获取，失败时使用默认皮肤或后台头像。

### 活跃度榜单

- 统计玩家在线时长、最近上线时间、活动参与次数、论坛互动、素材投稿、作品提交等活跃行为。
- 支持按周、月、季度、总榜展示。
- 支持公开榜单和成员内部榜单。
- 管理员可调整统计规则权重。

### 业绩考勤与贡献积分

- 玩家通过入服审核并加入服务器时，考勤积分初始为 60 分。
- 玩家完成工程、参与活动、提交有效作品、维护公共设施、提供宣传素材、协助管理等行为可以加分。
- 加分可来源于规则自动计算，也可由管理员手动添加。
- 如果玩家连续一个月没有上线、没有参与工程、没有参与活动，则扣 15 分。
- 当考勤积分小于等于 0 分时，玩家将进入移除白名单流程。
- 被移除白名单后如需重新加入，必须重新通过入服审核，且本次考核难度必须高于上次。
- 所有分数变动必须记录流水、原因、来源、操作人和时间。

### 验收标准

- 成员列表只展示允许公开的成员信息。
- 成员事迹公开前需要审核。
- 榜单数据需要标明统计周期。
- 自动扣分任务必须避免重复扣同一周期。
- 管理员手动加减分必须填写原因。
- 积分小于等于 0 后不能直接无记录删除白名单，必须进入可追溯处理流程。

## 5.10 活动、日历、更新日志与资源模块

### 功能说明

用于管理服务器活动、日程、版本更新和资源下载。

### 功能点

- 活动与赛事：建筑比赛、生存挑战、节日活动、PVP 赛事、探索任务等。
- 活动报名、参与名单、结果、获奖名单、奖励记录和贡献加分。
- 服务器日历：展示活动、维护、工程节点、投票截止、版本更新。
- 版本更新日志：记录服务器版本、插件变更、规则调整、资源包更新、地图更新。
- 资源管理：展示客户端整合包、材质包、光影包、地图文件、规则文档、活动资源。
- Cloudreve 集成：第一阶段可使用分享链接，后续可对接 API。
- 资源权限控制：公开资源、成员资源、管理员资源。

### 验收标准

- 已结束活动不能继续报名。
- 活动结果由管理员发布后才公开展示。
- 日历事件需要有来源和状态。
- 重要规则变更必须关联对应指南版本。
- 用户能清楚看到资源适用版本和更新时间。
- Cloudreve 无法访问时，官网需要提示资源服务暂不可用。

## 5.11 管理后台、审计、备份与数据看板模块

### 功能说明

管理后台是 owner、admin、helper 使用的运营工作台。它统一管理用户、邀请码、审核、素材、白名单、考勤、工单、举报、内容和配置。

### 功能点

- 用户管理：查看用户、修改角色、封禁账号、查看绑定信息。
- 邀请码管理：owner 可创建管理员邀请码，owner/admin 可创建玩家邀请码。
- 指南管理：维护服规、进服指南、游玩指南、玩家指南、工程项目、开放活动说明。
- 考核管理：维护红石、后期、建筑、其他方向题库和二次考核规则。
- 白名单管理：处理白名单申请、移除白名单、查看重新入服记录。
- 榜单管理：配置活跃度榜单、业绩考勤榜单、贡献分规则。
- 考勤管理：查看分数流水、手动加减分、处理积分小于等于 0 的成员。
- 内容管理：维护公告、作品、摄影、精彩瞬间、专题页、服务器进度、成就、里程碑。
- 素材审核：处理玩家上传的图片、视频、压缩包、文档等素材。
- 工单管理：分配、回复、关闭玩家工单。
- 举报与处罚管理：处理举报、登记处罚、公示或隐藏处罚记录。
- 系统配置：配置服务器地址、线路、开服时间、站点信息、多语言、主题、SEO、上传限制、外部交流入口、在线地图接口预留地址。
- 数据看板：查看注册、活跃、考核、白名单、素材、工单、举报、积分等数据。
- 审计日志：记录管理员关键操作。
- 备份与恢复：记录 MySQL、上传素材、关键业务数据的备份策略和恢复方案。

### 验收标准

- 后台只能 owner、admin、helper 访问，user 不能访问后台。
- owner/admin/helper 后台菜单和接口必须按权限限制。
- 删除、封禁、审核拒绝、移除白名单、手动扣分等高风险操作需要二次确认。
- 审核操作必须记录审核人、审核时间、审核结果和审核意见。
- 管理员关键操作需要记录审计日志。

## 6. 审核、状态与积分规则

## 6.1 通用内容状态

- 草稿：内容尚未发布。
- 待审核：玩家提交后等待管理员审核。
- 已通过：内容允许公开或按权限展示。
- 已拒绝：内容不展示，并记录拒绝原因。
- 需修改：管理员要求提交者修改后再次提交。
- 已下架：曾经发布过，但被管理员下架。
- 已归档：保留记录，但不作为主要内容展示。

## 6.2 上传素材审核流程

```text
玩家上传素材 -> 系统校验文件 -> 进入待审核 -> 管理员审核
  -> 通过：进入展示平台或素材库
  -> 拒绝：记录原因并通知玩家
  -> 要求修改：玩家重新提交
```

## 6.3 考勤积分流程

```text
玩家通过入服审核 -> 初始化 60 分
  -> 完成工程 / 参与活动 / 提交贡献 -> 自动或手动加分
  -> 连续一个月无上线、无工程、无活动 -> 扣 15 分
  -> 分数 <= 0 -> 待移除白名单
  -> 管理员确认 -> 移除白名单
  -> 玩家重新申请 -> 更高难度二次考核
```

## 7. 非功能需求

## 7.1 安全性

- 密码必须加密存储，不能明文保存。
- 注册必须校验邀请码，管理员邀请码必须与玩家邀请码隔离。
- 登录、注册、上传、投票、考试、工单、举报等接口需要限制频率。
- 管理后台接口必须做权限校验。
- 用户上传文件需要限制类型、大小、数量和频率。
- 上传文件需要做安全检查，禁止可执行脚本直接公开访问。
- 管理员操作需要记录审计日志。
- 高风险操作需要二次确认。

## 7.2 权限最小化

- 管理系统固定为 owner、admin、helper、user 四级权限。
- 管理员只能看到自己权限范围内的后台菜单和接口。
- helper 只能处理授权范围内的初审、回复、协助维护工作。
- user 不能访问后台接口。
- 手动加减考勤分、移除白名单、生成管理员邀请码、修改系统配置必须限制为 owner 或 admin，其中管理员邀请码和核心系统配置仅 owner 可操作。

## 7.3 性能

- 首页首屏加载应尽量控制在 3 秒内。
- 图片、视频封面需要压缩、缩略图和懒加载。
- 服务器状态查询应使用缓存。
- 论坛、作品、素材、成员、资源、工单、举报、榜单列表需要分页。
- 大文件上传应支持进度提示，后续可扩展分片上传。

## 7.4 可维护性

- 前后端分离开发。
- 后端接口返回统一数据格式。
- 后端按模块分层：控制器负责接请求，服务层负责业务逻辑，数据层负责数据库读写。
- 配置项集中管理，例如服务器地址、线路、开服时间、Cloudreve 地址、上传限制、积分规则、外部交流入口。

## 7.5 备份与可观测性

- MySQL 数据需要定期备份。
- 上传素材和资源元数据需要备份。
- 邀请码、白名单、考勤积分、处罚记录、审核记录属于关键数据，需要重点备份。
- 后端服务需要健康检查接口。
- 需要记录接口错误日志、接口耗时、上传失败、外部服务调用失败。
- 需要监控磁盘空间，避免素材上传导致磁盘写满。

## 12. 待确认问题

- 积分小于等于 0 后是否自动移除白名单，还是必须管理员确认。
- Cloudreve 是只使用分享链接，还是需要调用 Cloudreve API。
- 玩家上传素材是否直接存 Cloudreve，还是先存官网服务器再同步。
